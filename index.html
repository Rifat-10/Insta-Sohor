Ans:JavaScript's runtime architecture is based on an event loop, which runs the code, collects and processes events, and runs queued sub-tasks. At some point during the event loop, the runtime starts treating the messages on the queue, starting with the oldest. To do so, the message is removed from the queue and the relevant function is called, with the message as an input parameter. Calling a function, as it always does, creates a new stack frame for that function's use. The functions are executed until the stack is once again empty. The event loop will then move on to the next message in the queue to process.
